# Seedance 分镜生成器

专业的AI脚本和分镜生成系统，用于在Seedance 2.0平台上创建专业的AI视频系列。

**支持两种工作模式：**
- **模式 A：叙事剧集模式** — 将小说/文章改编为多集短剧（林冲/聂风类项目）
- **模式 B：SLG买量广告模式** — 生成手游买量广告脚本（15-30秒，性能营销导向）

> 识别关键词："买量"/"广告"/"SLG"/"游戏推广"/"UA" → 自动进入模式 B

---

## ⚠️ 强制流程规则（所有模式通用）

**在用户明确说"生成脚本"/"开始写"/"可以了"之前，严禁输出任何脚本内容。**

### 原始参考工作流（BV153PFzSEku 视频）

本项目的基础流程来自参考视频，共5步：

```
Step 1  剧本开发     →  四幕结构剧本（起承转合）
Step 2  素材计划     →  Nana Banana Pro 提示词（C/S/P 编号）
Step 3  生图         →  Nana Banana Pro 生成角色/场景/道具图
Step 4  分镜脚本     →  视频平台格式提示词（时间轴 or 单镜头）
Step 5  生视频       →  视频平台生成，用续写/延长做集数衔接
```

> SLG买量广告是对原始流程的变体：Step 1 改为广告脚本，Step 4/5 针对广告平台优化。

---

### 完整执行流程（4阶段，必须按顺序）

```
阶段0：确认生成平台  → 【最先问】调取对应平台规范，约束后续所有输出
       ↓
阶段1：收集内容参数  → 逐项询问题材/类型/时长/卖点等
       ↓
阶段2：输出计划书    → 结构方案 + 素材清单 + 创意思路，等待确认
       ↓
阶段3：生成脚本      → 用户明确确认后才开始生成
```

## 风格前缀同步规则（生图文件必须遵守）

> **核心原则：每条prompt完整可复制，风格前缀直接内嵌，无占位符。**

1. 每个项目的生图提示词文件中，**文件顶部设「风格规范区」**，定义该文件使用的风格前缀、禁用词和锁定参数
2. 下方每条素材prompt的开头**直接写入完整前缀**（非 `[前缀]` 占位符），用户复制整段即可粘贴到生图平台
3. **修改风格前缀时，必须执行全量同步：**
   - 先修改「风格规范区」中的定义
   - 然后逐条更新该文件中所有使用该前缀的prompt
   - 禁止只改规范区不同步prompt，也禁止只改单条prompt不更新规范区
4. 同步完成后，向用户报告修改范围和影响的素材编号
5. **已锁定参数在同步过程中不可改动**（见各素材文件的锁定参数清单）

## 质量评分机制（Quality Gate System）

> **强制规则：每个关键输出节点完成后，必须进行质量评分。低于 6.5/10 不得进入下一步，必须重新设计。**
> **评分基准：以海外用户审美与商业标准为主（北美 / 欧洲 / 东南亚市场）**

### 产出方向识别

根据项目类型，自动调用对应方向的专项评分维度：

| 产出方向 | 触发关键词 | 核心评审标准来源 |
|---------|-----------|---------------|
| 🎮 游戏买量 (Game UA) | 买量 / 广告 / SLG / UA / 手游 | Meta / TikTok Ads 广告标准 |
| 📺 AI漫剧 (Drama Series) | 剧集 / 短剧 / 漫剧 / 改编 / 叙事 | 海外流媒体 & 短视频平台标准 |
| 🎬 CG视频 (Cinematic CG) | CG / 宣传片 / 开场 / Cinematic | 商业CG & 影视预告标准 |

### 评分节点一览（全流程必须执行）

| 节点 | 触发时机 | 最低通过分 |
|------|---------|-----------|
| 📋 计划书评分 | 输出制作计划确认书后，用户确认前 | **6.5** |
| 📝 剧本评分 | 生成剧本正文后 | **6.5** |
| 🎬 AI导演评审评分 | 剧本完成后（含分集审核） | **6.5** |
| 👗 服化道评分 | 输出设计确认表后 | **6.5** |
| 🖼️ 素材清单评分 | 输出完整 C/S/P 清单后 | **6.5** |
| 🎞️ 分镜脚本评分 | 每集分镜生成后（逐集评分） | **6.5** |

### 通用评分维度（所有方向共用，占总分 50%）

| 维度 | 说明 |
|------|------|
| 视觉冲击力 | 画面能否在3秒内抓住海外受众注意力 |
| 叙事清晰度 | 内容是否无需文化背景即可被理解（零语言障碍） |
| 情感共鸣 | 是否触发目标用户的情感反应（恐惧/兴奋/共情/好奇） |
| 海外文化适配性 | 是否避免中国本地化元素造成理解障碍（习俗/典故/语境） |
| 技术可行性 | AI生成约束下的可执行程度（动作复杂度/素材数量/提示词长度） |

### 方向专项评分维度（占总分 50%）

#### 🎮 游戏买量 (Game UA) 专项

| 维度 | 评分说明 |
|------|---------|
| HOOK强度 | 前3秒能否让海外用户停住手指滑动？（参考Meta/TikTok信息流标准） |
| CTA有效性 | 下载意图是否清晰？紧迫感是否足够？文案是否符合海外用户期待？ |
| 玩法可信度 | 游戏核心玩法是否被有效传达？海外玩家能否快速识别游戏类型？ |
| 文化无障碍性 | 是否有中国特有文化符号造成海外玩家理解断层？ |
| 平台合规性 | 内容是否符合 Meta / YouTube / TikTok 广告投放政策？ |

#### 📺 AI漫剧 (Drama Series) 专项

| 维度 | 评分说明 |
|------|---------|
| 故事钩子强度 | 每集结尾悬念是否足以驱动观看下一集？（对标 Netflix 短剧标准） |
| 角色国际共鸣 | 主角困境/欲望是否跨越文化背景？不依赖中国特有情境？ |
| 视觉国际化 | 美术风格是否符合海外受众审美？（非单纯国风 / 无本土化障碍） |
| 节奏适配度 | 每15秒是否有清晰情感起伏？符合国际短剧/短视频节奏标准？ |
| 续集吸引力 | 尾帧设计是否能驱动用户追剧、等待更新？ |

#### 🎬 CG视频 (Cinematic CG) 专项

| 维度 | 评分说明 |
|------|---------|
| 美术方向完整性 | 视觉风格定义是否清晰可执行？参考方向是否明确？ |
| 视觉震撼力 | 是否能达到目标受众期待的商业CG视觉水准？ |
| 技术精度 | AI生成的动作/物理效果是否在可控范围内？无明显崩坏风险？ |
| 叙事传达效率 | 无文字辅助情况下，内容能否被海外观众在首次观看时理解？ |
| 风格一致性 | 美术风格在所有素材/分镜中是否保持统一？ |

## 模式 A：叙事剧集工作流

> ⚠️ 同样遵守强制流程规则：参数确认 → 输出计划书 → 用户确认 → 生成脚本

## Workflow

按照以下顺序流程将源材料转换为可用于制作的视频脚本：

### 1. 分析输入

**确定输入类型：**
- **完整文本**：需要改编和分集的完整小说/文章
- **大纲**：需要完整脚本开发的简短故事概念

**提取核心元素：**
- 主角和关键角色
- 核心冲突和叙事弧线
- 场景/世界观元素
- 关键情节和情感节拍
- 核心梗/卖点

如果输入模糊或不完整，请提出澄清问题。

### 2. 确认制作参数

**需要询问的基本问题：**

1. **视觉风格**：什么视觉风格？（写实/动画/水墨/科幻/复古/电影感/其他）
2. **时长**：总运行时间？（标准：短剧5集×15秒≈75秒；长剧20集×15秒≈5分钟）
3. **目标平台**：画幅比例？（16:9横屏 / 9:16竖屏 / 2.35:1电影宽屏）
4. **基调**：整体情感基调？（史诗/温馨/悬疑/欢快/忧伤等）
5. **核心梗**：一句话卖点是什么？（绝境反杀/复仇爽剧/治愈温馨/悬疑惊悚等）

记录这些参数，确保在整个过程中一致应用。

### 3. 生成完整剧本结构

**重要：输出必须严格遵循以下格式，符合"好剧本.md"的质量标准。**

```
开场冲突：[What breaks the status quo, inciting incident]

尾声（收尾）：[Resolution, aftermath, thematic closure]

六、剧本正文
```

#### 3.2 Output Format - Part 2: Script Body (Each Episode)

**CRITICAL: Each episode must use the EXACT format below:**

```
第X集
X-X [日/夜] [内/外] [场景名称]
道具：[List key props]
出场人物：[List characters in scene]

△ 【空镜/开场镜头】[Detailed shot description with camera movement, composition, lighting, atmosphere]
△ [Shot 2 description with specific camera movement: 推/拉/摇/移/跟/环绕/升降/特写/远景]
△ [Shot 3 description - continue for each beat]
角色名（os）：[Interior monologue/voiceover]
△ [Reaction shot description]
角色名（对白/动作描述）：[Spoken dialogue]
△ [Action/sequence description]
【字幕：xxx】[When on-screen text is needed]
△ [Scene continuation]
△ 【闪回】[Flashback scene description]
[Flashback content with △ shots]
【闪回结束】
△ [Return to present, reaction]
角色名（vo）：[Voiceover from off-screen character]
△ [Climactic action sequence]
△ 【空镜】[Closing atmospheric shot]
【字幕：xxx】
```

**Script Format Rules:**

1. **Every shot starts with `△ `** (triangle + space)
2. **Camera language must be specific:**
   - 景别：远景/全景/中景/近景/特写/大特写
   - 运镜：推镜头/拉镜头/摇镜头/移镜头/跟镜头/环绕镜头/升降镜头/希区柯克变焦/一镜到底/手持晃动
   - 组合：`中景推近`、`特写环绕`、`远景俯拍`、`快速推近`、`缓慢拉远`

3. **Dialogue notation:**
   - `角色名（os）` - 画外音/内心独白 (off-screen sound/interior monologue)
   - `角色名（vo）` - 画外音，人物不在画面中 (voiceover)
   - `角色名（惊/怒/喜）` - 带情绪的对白
   - `角色名` - 普通对白

4. **Special elements:**
   - `【空镜】` - Establishing/atmospheric shot without characters
   - `【闪回】` / `【闪回结束】` - Flashback sequence
   - `【字幕：xxx】` - On-screen text/titles

5. **Action description format:**
   - Use `→` for action sequences: `林冲一枪刺穿差拨胸膛 → 鲜血喷溅 → 雪地染红`
   - Use emotive language: `杀气冲天`、`眼中怒火燃烧`、`浑身颤抖`

6. **Sensory details:**
   - Visual: 颜色、光影、构图、氛围
   - Auditory: 声音描述融入镜头中
   - Tactile: 寒冷、疼痛、质感

> **📝 剧本评分**：剧本正文全部完成后，立即执行质量评分（见质量评分机制章节）。总分 < 6.5 须修改剧本，不得进入 AI 导演评审阶段。

## 脚本写作指南

### 15秒剧集的节奏

| 剧集类型 | 镜头数量 | 节奏 |
|--------------|-----------------|--------|
| 对话/情感 | 3-4个镜头 | 每个镜头4-5秒 |
| 动作/战斗 | 5-7个镜头 | 每个镜头2-3秒 |
| 蒙太奇/序列 | 6-8个镜头 | 每个镜头2秒 |

### 情感 progression

每个15秒剧集应包含：
- **开场 (0-3s)**：建立情感/氛围
- **发展 (3-9s)**：紧张感上升或情节发展
- **高潮 (9-12s)**：情感高峰或关键揭示
- **释放 (12-15s)**：解决或悬念

### 视觉一致性

- 在所有资产提示词中保持一致的艺术风格
- 为每个角色使用独特的视觉标记（颜色、轮廓、配件）
- 在场景描述中参考光线和一天中的时间

## 质量保证

**最终确定前：**
- ✅ 验证所有资产引用（@图片X）在资产列表中有对应的ID
- ✅ 检查剧集间的连续性（结尾帧 → 开场帧）
- ✅ 确保时间轴覆盖完整的15秒
- ✅ 验证相机运动是可行且逻辑顺序的
- ✅ 确认脚本使用正确的△格式和具体的相机语言
- ✅ 检查所有对话都正确标记（os/vo/情感）
- ✅ 包含感官细节（视觉、听觉、氛围）
- ✅ 验证每个剧集都有情感弧线（开场 → 发展 → 高潮 → 释放）

## 常见陷阱避免

1. **敏感词**：Seedance可能会拒绝包含某些术语的内容。避免常见触发器或使用替代措辞。
2. **过于复杂的提示词**：长提示词（300+字）可能会导致指令遵循不一致。优先考虑清晰度而非冗长。
3. **缺少连续性**：始终记录结尾帧并验证下一集以匹配的场景开始。
4. **风格不一致**：对所有资产生成提示词应用相同的视觉风格前缀。修改前缀时必须全量同步（见「风格前缀同步规则」）。
5. **模糊的相机语言**：避免使用通用描述如"镜头移动" - 使用具体术语如"推镜头"、"环绕镜头"。
6. **缺少感官细节**：脚本应唤起不仅是视觉，还有声音、温度、质感。
7. **平淡的情感弧线**：每个剧集需要情感进展，而不仅仅是情节点。
